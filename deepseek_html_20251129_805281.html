GitHub Pages部署指南

快速将你的HTML项目部署到免费的GitHub Pages

什么是GitHub Pages？

GitHub Pages是一项免费的静态网站托管服务，可以直接从GitHub仓库部署HTML、CSS和JavaScript文件。你可以使用GitHub Pages托管个人、项目或组织网站。

部署流程概览

准备HTML文件
创建GitHub仓库
上传文件到仓库
启用GitHub Pages
访问你的网站

详细步骤

准备你的HTML文件

确保你的HTML文件命名为index.html，这样它将被自动识别为网站的主页。

<!DOCTYPE html> <html> <head> <title>我的网站</title> </head> <body> <h1>欢迎访问我的网站！</h1> </body> </html>
注意： 如果你的项目有多个HTML文件，确保它们之间的链接使用相对路径。

创建GitHub仓库

登录你的GitHub账户，点击右上角的"+"图标，选择"New repository"。

  • 给仓库起一个名字（例如：my-website）
  • 选择公开（Public）或私有（Private）
  • 可选择添加README文件

上传文件到仓库

有几种方式可以将文件上传到GitHub仓库：

方法一：通过网页界面上传

在仓库页面，点击"Add file" → "Upload files"，然后将你的HTML文件和其他资源拖拽到上传区域。

方法二：使用Git命令行

# 克隆仓库到本地 git clone https://github.com/你的用户名/仓库名.git # 进入仓库目录 cd 仓库名 # 添加你的HTML文件 # 提交更改 git add . git commit -m "添加网站文件" # 推送到GitHub git push

启用GitHub Pages

在仓库页面，点击"Settings"标签，然后在左侧菜单中找到"Pages"。

  • 在"Source"部分，选择分支（通常是main或master）
  • 选择根目录（/ (root)）
  • 点击"Save"

几分钟后，你的网站将在以下URL上线：
https://你的用户名.github.io/仓库名

访问你的网站

在GitHub Pages设置页面，你会看到你的网站URL。点击该链接即可访问你部署的网站。

如果这是你的第一个用户网站，URL将是：
https://你的用户名.github.io

常见问题解答

我的网站没有显示最新更改？

GitHub Pages部署通常需要几分钟时间。如果更改没有立即显示，请等待几分钟后刷新页面。

我可以使用自定义域名吗？

是的，你可以在GitHub Pages设置中添加自定义域名。你还需要在域名提供商处配置DNS记录。

为什么我的网站显示404错误？

请确保：1) 你的仓库是公开的；2) 你已正确启用GitHub Pages；3) 你的主HTML文件名为index.html。

GitHub Pages支持PHP或其他服务器端语言吗？

不支持。GitHub Pages只能托管静态内容（HTML、CSS、JavaScript）。
